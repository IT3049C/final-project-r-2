## Short reflection

- **What we learned:** Building GameHub deepened our understanding of React Router for a multi-page feel inside one SPA, and of lifting shared state with context so the player name follows users across every game without prop drilling.
- **What we learned:** We also got practical experience wiring automated checks with Playwright (install hooks, stable selectors) so navigation and core flows stay trustworthy as we iterate.
- **What we would change next:** We would add more focused tests per game, run a structured accessibility pass (keyboard paths, contrast, labels), and consider small UX polish like local high scores or clearer win/loss feedback where it helps learning goals.
- **How peer feedback shaped decisions:** Classmates pushed us to keep navigation obvious (clear paths home, consistent header) and to make the first-run experience understandable—prompting for a name once and showing it in the shell—so the app feels cohesive rather than a loose bundle of demos.
- **How peer feedback shaped decisions:** Reviewers also encouraged us to prioritize testability (`data-testid` hooks, predictable layouts), which influenced how we structured components and landing content so flows could be exercised reliably in CI.
